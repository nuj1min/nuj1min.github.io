About

코드 너머, '사람'과 '시스템'을 잇는 개발자

"개발의 본질은 결국 '소통'과 '문제 해결'에 있으며
이것이 팀과 기술을 연결한다고 믿고 있습니다."

서버 인프라를 구축하고 트래픽을 제어하는 기술적 역량만큼이나, 제가 가장 중요하게 생각하는 것은 투명하고 유기적인 소통입니다. 양질의 결과물은 뛰어난 개인의 코드가 아닌, 한 몸처럼 움직이는 팀의 단단한 결속력에서 비로소 완성된다고 믿기 때문입니다. 다양한 조직에서 기획 및 리드 역할을 수행하며 대규모 인원의 의견을 조율하고, 팀원들이 하나의 목표를 향해 나아가도록 이끄는 협업의 가치를 깊게 체득해 왔습니다.

이러한 소통 능력의 기반에는 새로운 기술과 낯선 과제 앞에서도 절대 물러서지 않는 자신감과 열정이 있습니다. 프로젝트 중 한 번도 다뤄보지 않은 기술 스택이나 복잡한 요구사항이 주어지더라도 결코 두렵지 않습니다. 필요하다면 며칠 밤을 지새워서라도 기어코 원리를 파악하고 묵묵히 구현해 내고야 맙니다.

교내 행정 조교를 통한 행정 실무 및 학업, 매일 운동을 통해 빚어낸 단단한 체력과 끈기는 제가 수많은 협업을 성공적으로 완주할 수 있었던 가장 강력한 원동력입니다. 팀원들과 끊임없이 교감하며 최적의 아키텍처를 설계하고, 풀리지 않는 에러 앞에서도 결코 타협하지 않고 돌파해 내는 든든한 백엔드/클라우드 엔지니어 김민준입니다.

whatshu : GDG 출석체크 서비스
프로젝트 개요

GDG(Google Developer Groups) 캠퍼스 활동 중, 기존 구글폼 기반의 번거로운 행사 출결 프로세스를 자동화하기 위해 기획된 Spring Boot 기반 모바일 출석 관리 서버입니다. 백엔드 리드로서 전체 시스템 아키텍처 설계와 인프라 구축을 총괄했습니다.

Technical Stack

Java 17, Spring Boot 3.x, Spring Security, JPA/Hibernate, MySQL, Redis, AWS EC2, Docker, GitHub Actions

핵심 기술적 문제 해결
  • 보안과 UX를 고려한 인증 아키텍처: 단순 JWT 토큰의 탈취 위험을 방지하기 위해 Access/Refresh Token 전략을 도입했습니다. Redis를 활용하여 Refresh Token을 관리하고, 프론트엔드와 Silent Refresh 로직을 협의하여 사용자 재로그인 없이 보안을 유지하는 구조를 설계했습니다.
  • Docker 기반 인프라 경량화 및 CI/CD: AWS 프리티어 환경의 메모리 제한 문제를 해결하기 위해 Docker 멀티 스테이지 빌드를 적용하여 이미지 크기를 60% 이상 줄였으며, GitHub Actions를 통한 자동 배포 환경을 구축하여 운영 리소스를 최소화했습니다.
  • 협업 효율성을 위한 API 문서화 및 테스트: Swagger(SpringDoc)를 연동하여 프론트엔드 개발자가 실시간으로 API를 테스트할 수 있는 환경을 제공했습니다. 또한 JUnit5를 활용한 단위 테스트로 핵심 비즈니스 로직의 안정성을 확보했습니다.
성과 및 협업 포인트

데이터 정규화 기반의 효율적인 DB 설계로 중복 데이터를 제거하고 쿼리 성능을 최적화했습니다. 결과적으로 기존 수기 관리 대비 출석 체크 시간을 획기적으로 단축하며 팀 프로젝트의 기술적 기틀을 마련했습니다.
